type Rule[I, O any] struct {
    Select Action[I, O]   // Transform
    From   Source[I]      // Event source
    Where  If[I]          // Input condition
    Having If[O]          // Output condition
    Into   Destination[O] // Output
}
⚠️

The Problem

Event handling scattered across codebases. Difficult to test, hard to modify, impossible to compose. Side effects mixed with business logic. No reusability.

✨

The Solution

Declarative event pipelines with type safety. Composable components. Hierarchical rules with inheritance. Middleware for cross-cutting concerns.

Features

🔒

Type-Safe Pipelines

Generic types ensure compile-time correctness from source to destination. The compiler verifies your event flows.

🎯

Zero Dependencies

Any Go type works as an event. No interface requirements. Works seamlessly with existing types.

📋

Declarative Conditions

Boolean expressions for event filtering. Rich operators for numbers, strings, and collections.

🌲

Hierarchical Rules

SubRules inherit and extend parent configuration. DRY event processing patterns.

🔌

Unified Middleware

Single interface for callbacks, transformations, and destinations. Composable cross-cutting concerns.

⚡

Source Fanout

Shared sources start once and distribute events to all registered rules automatically.

Quick Start

Install via Go modules 
go get github.com/emad-elsaid/firehose

Hello World: Timer Events

package main

import (
    "context"
    "time"
    
    fh "github.com/emad-elsaid/firehose"
    "github.com/emad-elsaid/firehose/condition"
)

type Tick struct {
    Time time.Time
}

func (t Tick) Get(key string) (any, error) {
    if key == "hour" {
        return t.Time.Hour(), nil
    }
    return nil, fmt.Errorf("unknown: %s", key)
}

func main() {
    ctx := context.Background()
    
    rule := &fh.Rule[Tick, string]{
        ID:     "business_hours",
        Select: FormatTime{},
        From:   Timer{Interval: 1 * time.Second},
        Where:  condition.Cond[Tick]("hour >= 9 and hour < 17"),
        Into:   Printer{},
    }
    
    registry, _ := fh.AddRule(ctx, nil, rule)
    fh.Start(ctx, registry, nil)
    fh.Wait(registry, nil)
}

Core Concepts

01

Event Source

Produces events of a specific type. HTTP servers, message queues, timers, file watchers.

type Source[T any] interface
02

Condition

Optional filter evaluated against event attributes. Boolean expressions or custom logic.

type If[I any] interface
03

Transformation

Converts input events to output events. Business logic, enrichment, validation.

type Action[I, O any] interface
04

Destination

Handles output events. Database writes, API calls, message publishing, logging.

type Destination[T any] interface
